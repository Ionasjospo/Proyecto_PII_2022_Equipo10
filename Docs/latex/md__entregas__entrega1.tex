En esta primera instancia nos enfocaremos fuertemente en el modelado del problema. A continuación se detallan los artefactos entregables para la primera instancia del proyecto y el formato de cada uno de ellos.\hypertarget{md__entregas__entrega1_autotoc_md2}{}\doxysubsection{Medio de entrega}\label{md__entregas__entrega1_autotoc_md2}
Se entregará en una \href{https://webasignatura.ucu.edu.uy/course/view.php?id=288&section=1}{\texttt{ tarea de Web\+Asignatura}} un link al repositorio del equipo conteniendo todos los artefactos entregables.

Un integrante por equipo deberá completar la entrega.

{\bfseries{No se admitirán entregas fuera de fecha.}}

$\ast$$\ast$\+Sólo se admitirán entregas por Web\+Asignatura en la tarea provista con link a Git\+Hub y en la rama main\textsuperscript{1}.$\ast$$\ast$\hypertarget{md__entregas__entrega1_autotoc_md3}{}\doxysubsection{Estructura del repositorio}\label{md__entregas__entrega1_autotoc_md3}
El repositorio del equipo deberá organizarse de la siguiente manera\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{docs/}
\DoxyCodeLine{src/}
\DoxyCodeLine{  Program/}
\DoxyCodeLine{  Library/}
\DoxyCodeLine{  Test/}
\DoxyCodeLine{  ...}
\DoxyCodeLine{README.md}

\end{DoxyCode}


Todos los entregables que sean de documentación (diagramas, tarjetas, documentos, etc.) deberán entregarse dentro del directorio {\ttfamily docs/}.

Los proyectos de código fuente deberán crearse dentro del directorio {\ttfamily src} (como es habitual).

Deberá existir ademas un archivo README.\+md en la raiz del repositorio para incluír las notas del equipo. Véase \href{\#notas}{\texttt{ Notas}}.\hypertarget{md__entregas__entrega1_autotoc_md4}{}\doxysubsection{Entregables}\label{md__entregas__entrega1_autotoc_md4}

\begin{DoxyItemize}
\item {\bfseries{Tarjetas CRC}} de las clases, responsabilidades y colaboraciones identificadas para resolver el problema
\begin{DoxyItemize}
\item Formato\+: png, jpg, jpeg, pdf, html o markdown.
\end{DoxyItemize}
\item {\bfseries{Diagrama de clases}} (UML simple) con las clases del modelo identificadas
\begin{DoxyItemize}
\item Formato\+: png, jpg, jpeg, pdf, html o markdown.
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__entregas__entrega1_autotoc_md5}{}\doxysubsection{Fecha de entrega}\label{md__entregas__entrega1_autotoc_md5}
Véase \href{../README.md\#roadmap-y-entregables}{\texttt{ Tabla de Entregas}}.\hypertarget{md__entregas__entrega1_autotoc_md6}{}\doxysubsection{Notas}\label{md__entregas__entrega1_autotoc_md6}
Alentamos al equipo a que utilice el archivo de README en su repositorio para incluír notas de reflexión durante el desarrollo del proyecto. Estas notas pueden incluír\+:


\begin{DoxyItemize}
\item Qué desafíos de la entrega fueron los más difíciles
\item Qué cosas aprendieron enfrentándose al proyecto que no aprendieron en clase como parte de la currícula
\item Qué recursos (páginas web, libros, foros, etc) encontraron que les fueron valiosos para sortear los desafíos que encontraron
\item Y cualquier otro tipo de reflexión, material o comentarios sobre el trabajo en el proyecto.
\end{DoxyItemize}
