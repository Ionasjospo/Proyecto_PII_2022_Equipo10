~\newline


En esta última instancia terminaremos el desarrollo del nuestro chatbot por completo. Hasta el momento han desarrollado el modelo. Ahora agregarán todas las piezas restantes que rodean al {\itshape core}.\hypertarget{md__entregas__entrega3_autotoc_md17}{}\doxysection{Entregables}\label{md__entregas__entrega3_autotoc_md17}

\begin{DoxyItemize}
\item {\bfseries{Proyecto de C\#}} que incluye\+:
\begin{DoxyItemize}
\item código de la solución ({\ttfamily src/\+Library/})
\item código para iniciar el chatbot ({\ttfamily src/\+Program/})
\item casos de test (en uno o mas proyectos de test dentro del directorio {\ttfamily src/\+Test/})
\item documentación (generada con \href{https://www.doxygen.nl/index.html}{\texttt{ doxygen}})
\end{DoxyItemize}
\end{DoxyItemize}

$\ast$$\ast$\+\_\+\+Aclaración\+\_\+$\ast$$\ast$\+: En esta oportunidad sí deberán entregar {\ttfamily Program.\+cs} con código funcional que inicie el chatbot.

$\ast$$\ast$\+\_\+\+Recuerda\+\_\+$\ast$$\ast$\+: Como siempre, haremos especial incapié en las justificaciones de principios y patrones utilizados.

$\ast$$\ast$\+\_\+\+Recuerda\+\_\+$\ast$$\ast$\+: Existe un catálogo de patrones de diseño que es parte de la bibliografía del curso. Si no puedes acceder al libro, existen recursos online como \href{https://refactoring.guru/design-patterns}{\texttt{ refactoring.\+guru/design-\/patterns}}. Es importante que conozcas los patrones del catálogo y los utilices en tu solución.\hypertarget{md__entregas__entrega3_autotoc_md18}{}\doxysection{Fecha de entrega}\label{md__entregas__entrega3_autotoc_md18}
Véase \href{../README.md\#entregas}{\texttt{ Tabla de Entregas}}.\hypertarget{md__entregas__entrega3_autotoc_md19}{}\doxysection{Medio de entrega}\label{md__entregas__entrega3_autotoc_md19}
Se entregará en una \href{https://webasignatura.ucu.edu.uy/mod/assign/view.php?id=244382}{\texttt{ tarea de Web\+Asignatura}} un link al repositorio del equipo conteniendo todos los artefactos entregables. Si no se entrega link a un commit específico de la rama {\itshape master}, el equipo docente evaluará el último commit en {\itshape master} previo a la fecha de entrega.

Un integrante por equipo deberá completar la entrega.

{\bfseries{No se admitirán entregas fuera de fecha.}}

$\ast$$\ast$\+Sólo se admitirán entregas por Web\+Asignatura en la tarea provista con link a Git\+Hub y en la rama master\textsuperscript{1}.$\ast$$\ast$

Todos los entregables que sean de documentación (diagramas, tarjetas, documentos, etc.) deberán entregarse dentro del directorio {\ttfamily docs/}.

Los proyectos de código fuente deberán crearse dentro del directorio {\ttfamily src} (como es habitual).

Deberá existir ademas un archivo README.\+md en la raiz del repositorio para incluír {\bfseries{la descripción del proyecto}} y {\bfseries{las notas del equipo}}. Véase \href{\#notas}{\texttt{ Notas}}.\hypertarget{md__entregas__entrega3_autotoc_md20}{}\doxysection{Criterio de calificación}\label{md__entregas__entrega3_autotoc_md20}
Los profesores vamos a mirar el cumplimiento de los siguientes criterios, alineados con la rúbrica del curso\+:


\begin{DoxyItemize}
\item Aplica patrones GRASP
\item Aplica principios SOLID
\item Uso de C\#
\item Consistencia entre modelo UML/C\#
\item Aplica catálogo de patrones
\item Excepciones y diseño por contrato
\item Facilidad de comprensión/mantenimiento
\item Convenciones de código
\item Reutilización
\item Uso del repositorio
\item Unit tests de clases
\item Unit tests de escenarios
\item Funcionamiento del bot
\end{DoxyItemize}\hypertarget{md__entregas__entrega3_autotoc_md21}{}\doxysection{Notas}\label{md__entregas__entrega3_autotoc_md21}
Alentamos al equipo a que utilice el archivo de README en su repositorio para incluír notas de reflexión durante el desarrollo del proyecto. Estas notas pueden incluír\+:


\begin{DoxyItemize}
\item Qué desafíos de la entrega fueron los más difíciles
\item Qué cosas aprendieron enfrentándose al proyecto que no aprendieron en clase como parte de la currícula
\item Qué recursos (páginas web, libros, foros, etc) encontraron que les fueron valiosos para sortear los desafíos que encontraron
\item Y cualquier otro tipo de reflexión, material o comentarios sobre el trabajo en el proyecto.
\end{DoxyItemize}
